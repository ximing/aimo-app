# PRD: OCR 识别功能

## 简介

为 Aimo 应用添加 OCR (光学字符识别) 功能，允许用户从图片或 PDF 文件中提取文字内容。该功能集成在两个入口：
1. 浮动操作栏的"更多"菜单中 - 选择图片/PDF 后创建新 memo 并自动 OCR
2. 创建/编辑 memo 页面的底部工具栏中 - 在当前页面进行 OCR 并将结果填入输入框

## 目标

- 在浮动操作栏的"更多"菜单中增加 OCR 入口
- 在创建/编辑 memo 页面的底部工具栏增加 OCR 入口
- 支持从相机拍照、相册选择图片、文件管理器选择 PDF 文件
- 支持一次性选择多张图片或多页 PDF 进行批量 OCR 识别
- 文件先作为附件上传到服务器，生成 HTTPS 可访问的 URL
- PDF 文件直接作为附件保存，OCR 服务支持直接识别 PDF 链接
- OCR 完成后将识别文本（多图结果以换行分隔）自动填入 memo 输入框
- 提供加载状态和错误提示 UI

## 功能需求

- **FR-1:** 在 `MediaActionDrawer` 组件中增加"识图"操作选项
- **FR-2:** 点击识图后弹出图片来源选择菜单（相机/相册/文件）
- **FR-3:** 实现 `expo-document-picker` 用于选择 PDF 文件
- **FR-4:** 图片来源选择后，将选中的文件信息传递到新建 memo 页面
- **FR-5:** 新建 memo 页面自动上传附件（调用 `uploadAttachment`）
- **FR-6:** PDF 文件直接作为附件保存，OCR 服务支持直接传入 PDF 链接进行识别
- **FR-7:** OCR 识别完成后将识别文本填入 TextInput，多图结果以换行分隔（新建时替换，编辑时追加）
- **FR-8:** 在 create.tsx 底部工具栏增加 OCR 按钮，与现有拍照/图片/视频/录音按钮并列
- **FR-9:** OCR 处理过程中显示加载状态（禁用按钮并显示 ActivityIndicator）
- **FR-10:** OCR 失败时显示错误提示 toast，允许用户重试

## 非目标

- 不实现 OCR 服务端 API（后端已提供）
- 不实现图片编辑或裁剪功能
- 不支持从剪贴板粘贴图片进行 OCR
- 不支持 OCR 结果的手动校准编辑界面

## 设计注意事项

### UI 设计
- 使用统一的图标风格（lucide-react-native）
- OCR 按钮使用 `ImageText` 或 `ScanText` 图标
- 加载状态使用与录音转写类似的提示样式
- 错误提示使用与现有错误提示相同的样式

### 组件复用
- 考虑提取 OCR 选择逻辑为自定义 Hook `useOCR`
- 文件选择逻辑可复用现有 `useMediaPicker` 或创建 `useFilePicker`
- OCR 状态管理可在 Service 层或使用 useState

### API 集成
- 参考 `docs/api/ocr.md` 中的 OCR API 文档
- 使用 `POST /api/v1/ocr/parse` 接口
- 支持传入文件 URL（图片或 PDF 链接）
- OCR 供应商由后端服务决定，无需前端指定

## 技术注意事项

### 依赖项
- 确认 `expo-document-picker` 已安装，如未安装需添加到 package.json
- 确认 OCR API 已在 api 目录实现（如未实现需创建）

### 性能考虑
- 大文件上传显示进度
- 多文件 OCR 串行处理避免并发限制

### 错误处理
- 相机/相册权限被拒绝时的提示
- 文件上传失败的重试机制
- OCR 服务不可用或识别失败的错误提示
- 网络超时处理

## 成功指标

- 用户可以在 3 步操作内完成 OCR 识别并创建 memo
- OCR 识别成功率（文字清晰可识别）达到 95% 以上
- 单张图片 OCR 响应时间在 3 秒内（网络正常情况下）

## 技术说明

- OCR 供应商由后端服务决定，前端无需提供选择
- OCR 服务 API 文档见 `docs/api/ocr.md`
